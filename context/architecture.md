# Architecture Context

## Stack

| Layer | Technology | Role |
|---|---|---|
| Backend | Python + Flask | Serves API routes, file operations, integrations, and built React app |
| Frontend | React 19 + Vite | Primary Cinema Paradiso UI |
| Styling | Plain CSS with custom properties | Black/gold design system and responsive layouts |
| Metadata | Plex + TMDB APIs | Local metadata enrichment and online discovery |
| Source Search | Prowlarr API | Torrent/indexer search and release watchlist checks |
| AI | Ollama local API | Pick My Movie recommendations |
| Local Storage | JSON files | Config, user lists, edited collections, followed releases, TMDB cache |

## System Boundaries

- `app.py` owns Flask routes, config persistence, local file operations, Plex/TMDB/Prowlarr/Ollama integrations, and user JSON stores.
- `src/` owns the React application and all new UI work.
- `src/styles.css` owns the React visual system.
- `templates/index.html` is the legacy UI, available at `/legacy` during migration.
- `dist/` is generated by Vite and served by Flask when present.
- `data/` is persistent user data and must not be committed.
- `cache/` and `res_cache.json` are rebuildable local cache and must not be committed.
- `winapp/` is paused and out of scope.

## Storage Model

- `config.json`: local settings and integration secrets.
- `data/user_lists.json`: user-created movie lists.
- `data/user_collections.json`: user-edited TMDB collection overrides.
- `data/followed_releases.json`: followed releases and their watch status.
- `cache/tmdb_library_cache.json`: rebuildable TMDB movie detail cache.
- `cache/tmdb_collection_cache.json`: rebuildable TMDB collection cache.
- `res_cache.json`: rebuildable resolution probe cache.

## Auth and Access Model

The app has no user accounts. It is designed for local desktop/self-hosted use. Security is based on local machine trust plus server-side path safety checks for file operations.

## Invariants

1. File delete, rename, and move operations must stay inside configured movie library roots.
2. Delete defaults to Recycle Bin behavior where supported.
3. User data and cache must remain separate.
4. TMDB cache can be discarded; user lists, followed releases, and edited collections cannot.
5. Online discovery actions must remain archive-aware when ownership data is available.
6. Do not touch `winapp/` during web app development.
